Task#2: Conversion rate для Пятерочки

Описание данных:
В основе задания sample из результатов нескольких волн опроса, приближенный к структуре данных Ivanov Index:
●	Каждая строка = один ответ на один вопрос от одного уникального респондента (respond_gen_id). Респонденты не повторяются из волны в волну.
●	Вопросы бывают двух типов: single choice ([S8] и [Q11V2]) и multiple choice ([M5]).
●	В multiple choice вопросах один респондент может выбрать несколько вариантов ответа.

Вопросы:
●	[Q11V2] К какому из следующих интервалов Вы могли бы отнести доход на одного члена Вашей семьи в месяц в данный момент? (comment. Single choice вопрос. Ответ указан в столбце “answer”).
●	[M5] Какой из этих магазинов есть в пешей доступности от Вашего дома? Для магазинов крупного формата в транспортной доступности? (comment. Multiple choice вопрос. Ответ указан в теле формулировки после символов | или &. Не обращайте внимание на содержание варианта “прочее” в столбце answer в данном вопросе).
●	[S8] В каком магазине Вы покупали основную часть продуктов в последнем месяце? (comment. Single choice вопрос.) 


Задача:
Теперь посчитаем Conversion rate для одного из основных игроков - “Пятерочки”.

Conversion rate “Пятерочки” = количество Core customers пятерочки ([S8]) / количество респондентов, которые посещают и “Пятерочку”, и “Магнит” (в [M5] указали оба варианта).

comment: показатель является прокси на конкуренцию между игроками. Чем выше conversion rate, тем привлекательнее игрок для лояльных покупателей (с большей частотой покупок и большими суммами чеков). Данный показатель позволяет оценить рыночную позицию розничной сети в сегменте «магазинов у дома», который играет важную роль в формировании выручки крупных игроков.


Task#3*: Базовый классификатор Pdf-файлов

Задача: провести классификацию pdf отчетов с помощью API любой LLM-модели (API-ключи для бесплатных версий многих моделей можно сгенерировать на https://openrouter.ai/). Pdf - файлы для классификации будут приложены

Пример:
1)	Базовый промпт: 
Создать таблицу (dataframe) с информацией о следующем:
•	Автор отчета и его название
•	Тематика отчета (кратко, одним словом) 
•	Ключевые слова из текста отчета (примерно 10 штук) 
•	Краткий итог (summary) в двух-трёх предложениях

2)	Желаемый output:
Автор и название	Тематика	Ключевые слова (≈10)	Краткое summary (2–3 предложения)
IPSOS; Здравоохранение России глазами пациента 	Здравоохранение	ОМС; ДМС; ПМУ; диспансеризация; врач; диагностика; электронная карта	Отчёт фиксирует, что пациенты видят в ОМС надёжный, но медленный «базовый слой» и компенсируют его недостатки платными услугами или ДМС. 

note: формат таблицы можно незначительно изменить, но структура должна строго соответствовать заданному системному промпту.
